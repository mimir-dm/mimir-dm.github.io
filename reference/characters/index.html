Mimir - D&D Campaign Assistant

Character and NPC Management

Mimir provides comprehensive tools for managing player characters (PCs) and non-player characters (NPCs) within your campaigns.

Overview

The character system supports:

  • Player Characters (PCs): Full character sheets with class features, spells, and inventory
  • Non-Player Characters (NPCs): Quick stat blocks for encounters
  • Character Versioning: Track character progression over time
  • Spell Management: Known spells, prepared spells, and spell slot tracking

Data Model

Players

Players represent the real people at your table. Each player can have multiple characters across different campaigns.

Player
├── id
├── name
├── email (optional)
└── characters[] (via campaign_players)

Characters

Characters are the in-game personas controlled by players (PCs) or the DM (NPCs).

Character
├── id
├── player_id (null for NPCs)
├── character_name
├── race, class, level
├── ability_scores (STR, DEX, CON, INT, WIS, CHA)
├── hit_points (current, max, temp)
├── armor_class
├── speed
├── proficiencies
├── features
├── spells (known, prepared, slots)
└── inventory

Character Creation

Creating a PC

PCs are linked to a player and campaign:

  1. Create or select a player
  2. Choose race and class from the catalog
  3. Set ability scores (manual entry or standard array)
  4. The system calculates:
    • Hit points based on class and CON modifier
    • Proficiency bonus based on level
    • Spell slots based on class and level
    • Starting equipment suggestions

Creating an NPC

NPCs are simpler and don’t require a player:

  1. Provide basic stats (name, race, class)
  2. Set ability scores
  3. Optionally add spells and features
  4. NPC is created with is_npc: true

Spell Management

Spell Slots

The system automatically calculates spell slots based on:

  • Character class
  • Character level
  • Multiclass combinations (if applicable)

Supported caster types:

  • Full Casters: Wizard, Cleric, Druid, Bard, Sorcerer
  • Half Casters: Paladin, Ranger
  • Third Casters: Eldritch Knight, Arcane Trickster
  • Pact Magic: Warlock (separate slot progression)

Spell Operations

OperationDescription
add_spell_to_knownAdd a spell to character’s known spells
prepare_spellsSet which known spells are prepared
cast_spellCast a spell, consuming a slot
take_restShort or long rest to recover slots

Character Progression

Leveling Up

The level_up_character command handles:

  • Incrementing character level
  • Recalculating hit points
  • Updating proficiency bonus
  • Adjusting spell slots
  • Prompting for ASI/feat choices (at appropriate levels)

Character Versions

Each significant change creates a version snapshot:

  • Level ups
  • Major stat changes
  • Class feature acquisitions

Retrieve history with get_character_versions.

Frontend Integration

The frontend provides:

  • Character creation wizard with step-by-step guidance
  • Character sheet view with all stats and abilities
  • Spell management interface with drag-and-drop preparation
  • Quick actions for common operations (cast, rest, level up)

LLM Tool Integration

The AI assistant can help with character management through natural language:

  • “Create a level 5 dwarf cleric named Thorin”
  • “What spell slots does my wizard have left?”
  • “Cast fireball at 4th level”
  • “Take a long rest”

See LLM Tools for more details on AI-assisted character management.
